# 问题追踪报告：DOCX 第3页内容溢出（多2-3行）

## 问题描述

| 项目 | 原PDF 第3页 | DOCX 第3页 |
|------|------------|-----------|
| 末行内容 | "三、获取采购文件" | "时间：2026年..." / "12:00至23:59..." / "地点：政采云平台线上" |
| 正文行数 | 28 行 | 31 行 |
| 多出 | — | **2-3 行**（属于原PDF第4页的内容） |

每页都有类似偏差，且随页数增加累积。

## 根因

**下边距检测把页脚行纳入了统计，导致内容区被错误地放大。**

### 错误链路

```
_detect_page_margins()
  → 遍历每页所有 span（包括页脚"—1—"）
  → max(y1s) 取到了页脚的 y1=794.7pt
  → ph - 794.7 = 47.2pt 作为"下边距"
  → 实际正文下边距应为 ~84pt
  → DOCX 内容区高度比 PDF 多出 ~37pt
  → 每页多放 1.5-2 行内容
```

### 精确数据

| 页 | 正文末行 y1 | 页脚 y1 | 正文下边距 | 页脚下边距 | 当前检测值 |
|----|------------|--------|-----------|-----------|-----------|
| 3  | 732.6 | 794.7 | 109.3 | **47.2** | 47.2 ← 被页脚污染 |
| 4  | 758.0 | 794.7 | 83.9 | **47.2** | 47.2 |
| 5  | 761.0 | 794.7 | 80.9 | **47.2** | 47.2 |
| 7  | 763.3 | 794.7 | 78.6 | **47.2** | 47.2 |

- 所有页的页脚"—1—"固定在 y1=794.7pt（下边距 47.2pt）
- 正文末行的下边距在 78-110pt 之间波动（内容最满的页约 79-84pt）
- 当前 `_detect_page_margins` 用 `min(bottom_margins)`，而每页的 `max(y1s)` 都包含了页脚，所以检测值恒为 47.2pt
- **正确下边距应为 ~84pt**（内容最满的页反映真实值）

### 影响量化

```
DOCX 当前下边距: 47.2pt
PDF 真实下边距:  ~84pt
差值:            ~37pt
每行高度:        ~21pt
多放行数:        37 / 21 ≈ 1.8 行/页
```

第3页恰好因为上方累积也有微小偏差，最终多放 2-3 行。

## 次要因素（已排除）

### 非根因：行高计算

`bbox高度/行数` 方案的行高在 LibreOffice 渲染验证下基本准确。逐行 y 坐标对比显示 28 行内累积偏差仅 +1.3pt（第1行 +1.0pt → 第28行 +1.3pt），**行高计算不是溢出主因**。

### 非根因：换行数差异

DOCX 内容区宽度（420pt）≥ PDF 正文行宽（409pt），不会因列宽变窄而多换行。实际渲染确认第3页 28 行内容在 DOCX 中也是 28 行。

## 涉及代码

| 文件 | 函数 | 行号 | 问题 |
|------|------|------|------|
| `build_docx.py` | `_detect_page_margins` | ~1195-1258 | 未排除页脚行，`max(y1s)` 包含页脚坐标 |

## 修复方向

在 `_detect_page_margins` 中排除页脚行后再统计 `max(y1s)`。

页脚的特征：`y0 > 页高 - 60`（页脚固定在页面底部约 47pt 处）。排除页脚后，用内容最满的页（`min(bottom_margins)`）的正文末行 y1 来计算真实下边距。

> 注意：上边距检测同理可能被页眉污染，但当前 PDF 无页眉，暂无影响。对称处理更健壮。
