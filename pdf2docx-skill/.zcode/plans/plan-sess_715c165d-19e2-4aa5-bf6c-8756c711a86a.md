
# 降级 MinerU 3.0.4 → 2.7.6 方案

## 背景与原因

MinerU **3.0.4** 在你的环境（Windows + CPU，无 NVIDIA CUDA）上有**已知死锁 bug**（[Issue #4741](https://github.com/opendatalab/MinerU/issues/4741)、[#4882](https://github.com/opendatalab/MinerU/issues/4882)）：
- 任务能提交（202 Accepted），但 worker 在模型加载阶段 hang 住
- 内存稳定不增长（550MB 卡死）、CPU 停止消耗——不是慢，是死锁
- 这是 3.x 新架构 async 任务模式的原生层 bug，等再久也不会出结果

3.0.4 已卸载。现需降级到最后的稳定 2.x 版本 **2.7.6**（2026-02-06 发布，Windows+CPU 验证过可用）。

## 关键事实（已查证）

1. **2.x 的 API 不是 `UNIPipe`/`DiskReaderWriter`** —— 那是 1.x 旧 API。2.x 虽然包名可能保留 `magic_pdf` 内部模块，但官方推荐用 `do_parse` 函数（来自 [Discussion #5076](https://github.com/opendatalab/MinerU/discussions/5076)）
2. **2.x 的 `do_parse` 是同步调用** —— 不走 async server，直接函数调用返回结果，**避开了 3.x 的死锁 bug**
3. **真实 API 签名必须装上后实测确认** —— 网络上的文档/博客要么是 1.x 的要么是 3.x 的，2.7.6 的精确用法需要看实际源码

## 实施步骤

### 第1步：安装 2.7.6（需配国内源加速）

```bash
uv pip install "mineru[core]==2.7.6" -i https://pypi.tuna.tsinghua.edu.cn/simple
```
- 用清华源解决安装慢的问题
- 用 `[core]` 而非 `[all]`，只装 pipeline 后端（CPU 适用），减少体积

### 第2步：inspect 2.7.6 的真实 API（关键）

装完后直接探查实际可用的模块和函数：
```bash
# 看 do_parse 的真实签名
uv run python -c "from magic_pdf.pipe.UNIPipe import UNIPipe; help(UNIPipe.__init__)"
# 或
uv run python -c "import magic_pdf; print(dir(magic_pdf))"
```
这一步决定 `parse_mineru.py` 怎么写——**不靠猜，看实际源码**。

### 第3步：重写 `parse_mineru.py`

把当前基于 3.x `LocalAPIServer` + async HTTP 的实现，改为 2.x 的**同步 `do_parse`** 调用：
- 删除 `_import_mineru_sdk()`、`_parse_async()`、`LocalAPIServer` 相关逻辑
- 改为直接调用 `do_parse()`（同步函数，内部完成解析，输出 middle.json 到磁盘）
- 对外接口 `parse_with_mineru()` 的签名（入参、返回值）**保持不变**，这样 `pdf2docx.py` 和其他模块不用改

### 第4步：模型路径配置

已下载的 pipeline 模型在 `C:\Users\15927\.cache\modelscope\models\OpenDataLab--PDF-Extract-Kit-1.0\`。
2.7.6 的模型配置方式可能与 3.x 不同（3.x 用 `mineru.json`，2.x 可能用 `magic-pdf.json` 或环境变量）——装完后实测确认。

### 第5步：用前 1-2 页小任务验证

```bash
uv run python scripts/parse_mineru.py "你的.pdf" -o ./_test/ --start 0 --end 2
```
确认：
- 不再死锁（同步调用能正常返回）
- 产出 middle.json
- 能被 `pdf_inspect.py middle` 正确读取

### 第6步：更新文档

- `references/architecture.md`：修正 2.x API 说明
- `requirements.txt`：固定 `mineru[core]==2.7.6`
- `setup.sh`：适配 2.x 的检测和配置

## 风险与应对

| 风险 | 应对 |
|------|------|
| 2.7.6 的 API 签名与预期不同 | 第2步先 inspect 实际 API，再写代码 |
| 2.7.6 在 Windows 也有兼容问题 | 先只装 `[core]`（最小集），1-2 页小任务验证后再展开 |
| 已下载的 3.x 模型可能不兼容 2.x | 2.7.6 用 `mineru-models-download` 重新下载（已有的 modelscope 缓存可能可复用） |

## 不会改动的部分

以下模块**完全不受影响**，降级只动 MinerU 解析这一步：
- `parse_pymupdf.py`（字符级样式提取）—— 已验证可用
- `align.py`（bbox 对齐）—— 已验证可用
- `build_docx.py`（DOCX 重建）—— 已验证可用
- `pdf_inspect.py`、`pdf2docx.py` 主体 —— 对外接口不变
